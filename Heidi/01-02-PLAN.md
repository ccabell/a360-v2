## OBJECTIVE

A public, hardened SSE endpoint `POST /api/ask` that wraps the existing retrieval pipeline without modifying it: anonymous sessions, Supabase-native rate limiting, 24h answer cache, full ask logging, honest out-of-scope handling.

## HARD CONSTRAINTS

1. `lib/retrieval/*` and `app/api/research/chat/route.ts` remain untouched. The wrapper **imports and calls** the same functions the research route uses (read that route to see the exact entry points — likely `lib/retrieval/stream.ts` / `sources.ts` / `post-process.ts`) and re-emits the identical `ResearchEvent` SSE contract.
2. Service keys server-side only (this is a `route.ts`, so fine). No new client-side env.
3. No new infrastructure (no Upstash/KV) — Decision D4: rate limiting and caching are Supabase queries on `ask_log`.
4. Model budget — Decision D7: if the retrieval entry point accepts a model/config parameter, pass the cheaper gateway model for this route; if it doesn't accept one, DO NOT modify it — note the limitation in the completion report instead.

## TASKS

### T1 — Migration on Agent Manager Supabase (`aejskvmpembryunnbgrk`)
Apply via Supabase MCP, migration name `create_ask_log`:

```sql
create table if not exists ask_log (
  id uuid primary key default gen_random_uuid(),
  session_id text not null,
  surface text not null default 'public',          -- public | embed | dashboard
  question text not null,
  question_hash text generated always as (md5(lower(trim(question)))) stored,
  answer_prose text,
  citations jsonb,
  evidence_link_ids uuid[],
  status text not null default 'complete',         -- complete | error | rate_limited | out_of_scope
  latency_ms int,
  client_ip_hash text,
  created_at timestamptz not null default now()
);
create index if not exists ask_log_session_created_idx on ask_log (session_id, created_at desc);
create index if not exists ask_log_ip_created_idx     on ask_log (client_ip_hash, created_at desc);
create index if not exists ask_log_qhash_idx          on ask_log (question_hash, created_at desc) where status = 'complete';
```

### T2 — Anonymous session
`lib/ask/session.ts`: read cookie `a360_ask_sid`; if absent, generate `crypto.randomUUID()` and set it (httpOnly, sameSite=Lax — **but** `SameSite=None; Secure` when the request is for surface `embed`, else third-party iframe cookies die). Helper returns `{ sessionId, setCookieHeader? }`.

### T3 — The route
`app/api/ask/route.ts` (`POST`, body `{ question: string, surface?: 'public' | 'embed' }`):

Order of operations:
1. **Sanitize**: trim; reject empty; cap 500 chars (413-style friendly SSE error event, status `error`); strip HTML tags.
2. **Demo bypass**: if header `x-demo-token` === `process.env.DEMO_BYPASS_TOKEN` (non-empty), skip step 3.
3. **Rate limit** (two SQL counts): session > 10 in last hour, or ip_hash > 30 in last 24h → log a `rate_limited` row → emit a single SSE `error` event with `code: 'rate_limited'` and a friendly message ("You've hit the free limit for now — full access is available inside A360.") and `done`. `client_ip_hash` = sha256 of (`x-forwarded-for` first hop + `process.env.IP_HASH_SALT`). Never store raw IP.
4. **Cache check**: latest `ask_log` row where `question_hash` matches, `status='complete'`, `created_at > now() - interval '24 hours'`, `answer_prose is not null` → replay it as a well-formed SSE sequence (`status` "cached" → `sources` reconstructed from citations jsonb → full prose as one `token` event → `citations` → `done`). Log a new row (`status='complete'`, `latency_ms` actual, citations copied) so analytics count cache hits as asks.
5. **Live path**: call the shared retrieval entry exactly as the research route does; pipe its SSE through unchanged. Tee the stream server-side to capture final prose + citations.
6. **Out-of-scope** (Decision D5): if the `sources` event carries zero sources, do NOT stream a hallucination-prone answer — short-circuit: emit a `token` event with the honest template ("That's outside the A360 Global Library right now — it covers FDA-cleared aesthetic treatments. Closest topics I can answer on:") and a `citations` event with empty array, then `done`. Log `status='out_of_scope'`. Nearest-topic chips: 3 category names from a static import of the GL categories list (`lib/ask/nearest-topics.ts`, hardcoded from the live `categories` table — Neurotoxins, Dermal Fillers, Energy-Based Skin Tightening, Body Contouring, Weight Management; pick 3 by trivial keyword overlap, fall back to first 3).
7. **Log** the completed row (question, prose, citations, evidence_link_ids if surfaced by the pipeline, latency, surface, session, ip_hash).

## VERIFICATION

1. `curl -N -X POST localhost:3000/api/ask -H 'content-type: application/json' -d '{"question":"What are the contraindications for Botox Cosmetic?"}'` → SSE: `status` → `sources` (≥1) → `token`* → `citations` → `done`
2. Same curl again within a minute → first event is `status` "cached", full answer arrives near-instantly
3. Out-of-scope: `{"question":"What is the best mortgage rate in Boston?"}` → honest template, `citations: []`, DB row `status='out_of_scope'`
4. Loop 11 asks same session cookie → 11th gets `rate_limited` SSE error; row logged
5. Repeat with `x-demo-token: $DEMO_BYPASS_TOKEN` → no limit
6. SQL spot-checks: `select status, count(*) from ask_log group by 1;` shows all four statuses exercised; `question_hash` populated; no raw IPs anywhere (`select client_ip_hash from ask_log limit 3` shows hashes)
7. `/dashboard/research` still works (it doesn't touch this route, but confirm no shared-import side effects)
8. `git diff main --stat`: no changes under `lib/retrieval/` or `app/api/research/`
