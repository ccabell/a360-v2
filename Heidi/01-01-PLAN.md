## TASKS

### T1 — Create the shared component
`components/ask/ask-experience.tsx`

```tsx
export type AskVariant = 'dashboard' | 'public' | 'embed'

export interface VerbGroup {           // suggestion chips config (used by 01-03)
  verb: string                          // "Compare" | "Safety" | ...
  items: { query: string }[]
}

export interface AskExperienceProps {
  endpoint: string                      // '/api/research/chat' (dashboard) | '/api/ask'
  variant: AskVariant
  initialQuery?: string                 // from ?query= (01-03)
  autoSubmitInitial?: boolean
  suggestions?: VerbGroup[]             // rendered only when no answer yet
  showSave?: boolean                    // true only on dashboard
  placeholder?: string
  // embed bridge hooks (no-ops unless provided — wired in 01-04):
  onAskSent?: (m: { messageId: string; ts: number }) => void
  onAnswerComplete?: (m: { messageId: string; citationsCount: number }) => void
  onCitationClick?: (m: { url: string; tier: string }) => void
}
```

Move INTO this component, verbatim where possible: input state, SSE reader, event assembly, citation post-process call, answer rendering tree, error/loading states. Variant differences are *props and chrome only*:
- `dashboard`: `showSave` true, current paddings, current placeholder
- `public`: no Save; suggestions visible pre-ask; slightly larger initial type treatment (01-03 styles)
- `embed`: no Save; compact spacing class on the wrapper; transparent background

Suggestion chips live in a sibling: `components/ask/ask-suggestion-chips.tsx` (verb row → reveals 2–3 question chips; clicking a question chip calls the same submit path). Build it now, exercised in 01-03.
