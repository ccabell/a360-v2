## OBJECTIVE

The public face of A360 Evidence: an ungated, full-bleed page at `/ask` mounting `<AskExperience variant="public" endpoint="/api/ask">` with suggestion chips, `?query=` seeding, disclaimer, and tier legend — excluded from the password gate without weakening the dashboard.

## HARD CONSTRAINTS

1. shadcn/ui + OKLch tokens only (`bg-background`, `text-foreground`, `border-border`, `bg-card`, `text-muted-foreground`, `bg-accent`). Zero raw hex.
2. No Heidi strings, assets, or copy anywhere. All copy below is the approved original copy — use it verbatim.
3. The gate change must be a narrow allowlist, not a logic rewrite.
4. Both light and dark mode must pass — this page is the company's public face on June 22.

## TASKS

### T1 — Gate exclusion
Locate the shared-password protection (likely `middleware.ts` matcher or a check in a root layout). Add a public-path allowlist: `/ask`, `/embed`, `/api/ask`, plus existing static/asset exemptions. Touch nothing else about the auth flow.

### T2 — Bare layout + page
- `app/ask/layout.tsx`: minimal — html/body classes consistent with root, **no sidebar, no dashboard header, no footer** (Heidi suppresses its footer on this route for a reason: the experience is the page).
- `app/ask/page.tsx`: server component reading `searchParams.query`, rendering:

```
[A360 wordmark, small, top-center]                  ← reuse existing logo asset

H1:  Ask anything about aesthetic medicine.
Sub: Grounded answers from the A360 Global Library. Every claim cited. Free.

<AskExperience
  variant="public"
  endpoint="/api/ask"
  initialQuery={query}
  autoSubmitInitial={Boolean(query)}
  suggestions={ASK_SUGGESTIONS}
  placeholder="e.g. Can Botox and filler be done the same day?"
/>

Tier legend (xs, muted, above References once an answer exists — pass as prop or render slot):
  "Color = source authority:  ● FDA   ● Manufacturer   ● Peer-reviewed   ● Industry"
  (dots use the existing tier colors via the badge component, not hex)

Disclaimer (xs, muted, persistent at page bottom):
  "A360 Evidence provides educational information for licensed aesthetic
   professionals. It is not medical advice and is not a substitute for clinical
   judgment or the FDA-approved product label."
```

Initial state is centered hero (input vertically ~38% viewport); once first answer streams, layout shifts to a standard top-anchored conversation column (max-w ~768px). Match the dashboard's existing answer column width.

Chips UI (built in 01-01's `ask-suggestion-chips.tsx`): one row of verb chips (`rounded-md border border-border bg-background px-3 py-1.5 text-sm text-muted-foreground hover:text-foreground hover:bg-accent`); tapping a verb reveals its question chips beneath; tapping a question submits it. Chips disappear after first ask.

### T4 — Input behavior parity with the proven pattern
In the public variant: autofocus the input on desktop only (`window.matchMedia('(max-width: 768px)').matches` → skip), Enter submits, submit button disabled until trimmed input non-empty, spinner replaces the arrow while streaming, `aria-live="polite"` on the streaming region, visible focus rings.

### T5 — Rate-limit & out-of-scope cards
Render the `rate_limited` SSE error as a friendly inline card (message from the API + a ghost button "See A360 for practices →" linking `/`); render `out_of_scope` answers with the 3 nearest-topic chips as tappable follow-ups.
