Log every query in PAIRING_CORPUS_QUERY_LOG.md with: batch, pair, RPC, query string, result count, top chunk IDs.

**B) LLM 8-gate evaluation:**

For each pair, construct a prompt following the structure from RESEARCH.md:

System prompt: "You are evaluating a product pairing for a medical aesthetics platform. Be accurate, conservative, and evidence-based. Cite sources."

User prompt includes:
- Product A structured data: name, category, item_concerns list, does_not_solve array, key dossier fields
- Product B structured data: same
- SQL pre-screen signals: shared_concerns count and names, limitation coverage signals
- Corpus evidence chunks (top 3-5 most relevant per RPC)
- The PHASE_6_ANSWERS_PODCAST_SOURCED.md evidence for this pair (if exists)
- Known flags from D-21 (if applicable to this pair)

Required output format per gate:
- Gate N: PASS/FAIL
- Notes: [free-text explanation]

Then:
- recommended_tier
- relationship_type (from enum: complementary/stacks_with/sequential/enhances/alternative/contraindicated)
- relationship_strength (from enum: strong/moderate/weak)
- clinical_rationale (1-2 sentences)
- timing_guidance (1 sentence)
- same_session_ok (true/false with brief justification)
- patient_education_text (for canonical/common only, "Tell the patient: ..." format per D-14)
- staff_talking_points (for canonical/common only, education-first per D-13)

Use Anthropic SDK direct (not AI gateway per MEMORY). Model: claude-sonnet-4-20250514 for cost efficiency on high-volume evaluation.

**C) Apply hard stop rules:**
- If Gate 5 (Safety) = FAIL: max tier is conditional (with explicit conditions) or do_not_market/rejected
- If corpus returns 0 evidence across all 4 RPCs: max tier is conditional (flag LOW_CONFIDENCE)
- If "why both?" is circular or promotional: rejected or do_not_market
