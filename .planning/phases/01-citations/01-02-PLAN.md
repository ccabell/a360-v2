**Step 1 — Query evidence_links to get offering_ids per product (per D-02):**
Use Supabase MCP `execute_sql` on project `aejskvmpembryunnbgrk`:
```sql
SELECT DISTINCT el.offering_id, p.name as product_name
FROM evidence_links el
JOIN products p ON p.id = el.offering_id
WHERE el.source = 'fda_label'
ORDER BY p.name;
```
This gives the exact offering_id → product_name mapping needed for the URL table.

**Step 2 — Research FDA URLs for each product:**
For each product, use WebFetch to look up the FDA Access Data page. The lookup approach:
- Use `https://www.accessdata.fda.gov/scripts/cder/daf/index.cfm?event=overview.process&ApplNo={BLA_number}` to find the approved labeling page
- Or construct the URL directly: `https://www.accessdata.fda.gov/drugsatfda_docs/label/{year}/{BLA}s{supplement}lbl.pdf`
- Verify each URL resolves (HTTP 200) by fetching the URL

Known BLA numbers:
- Botox Cosmetic: 103000 (verified: `https://www.accessdata.fda.gov/drugsatfda_docs/label/2023/103000s5326lbl.pdf`)
- Xeomin: 125360
- Skinvive: 761261
- Juvederm products: 125511 (different supplements per variant)
- Kybella: 208009
- Dysport: 125274

**Note on Juvederm:** All Juvederm variants likely share BLA 125511 but have different supplement numbers. Some may share the same combined labeling PDF. Research each individually; if they share a PDF, record the same URL for all.

**Step 3 — Create `scripts/backfill-fda-urls.ts`:**
1. Loads env from `.env.local` using `dotenv.config({ path: path.join(process.cwd(), ".env.local") })`.
2. Creates Supabase client: `createClient(process.env.NEXT_PUBLIC_AGENT_SUPABASE_URL!, process.env.AGENT_SUPABASE_SERVICE_KEY!)`.
3. Defines a `FDA_URL_MAP: Record<string, string>` mapping `offering_id` to the researched FDA URL. Hardcode all 13 entries. For any product where the URL could NOT be determined, set value to empty string `""` and add a comment.
4. For each entry in FDA_URL_MAP where url is non-empty:
   ```typescript
   await agentDb
     .from("evidence_links")
     .update({ url: fdaUrl })
     .eq("source", "fda_label")
     .eq("offering_id", offeringId)
     .or("url.is.null,url.eq.");
   ```
5. Tracks: `updated: N, skipped (no URL researched): M, already had URL: K`.
6. Prints gap report at end: lists any products where URL was set to empty string (Chris fills manually per D-02).

**Step 4 — Run the script:** `npx tsx scripts/backfill-fda-urls.ts` — capture output.
