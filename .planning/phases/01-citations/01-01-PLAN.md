**Part B — PubMed backfill script (per D-04, D-05, D-06):**
Create `scripts/backfill-pubmed.ts` that:

1. Loads env from `.env.local` using `dotenv` with explicit path: `dotenv.config({ path: path.join(process.cwd(), ".env.local") })`.
2. Creates Supabase client: `createClient(process.env.NEXT_PUBLIC_AGENT_SUPABASE_URL!, process.env.AGENT_SUPABASE_SERVICE_KEY!)`.
3. Queries: `SELECT id, doi, pmid, url FROM evidence_links WHERE source = 'pubmed' AND pmid IS NULL AND doi IS NOT NULL`.
4. For each row, calls `https://api.crossref.org/works/${encodeURIComponent(doi)}` with header `User-Agent: A360-citation-backfill/1.0 (ccabell@aesthetics360.com)`.
5. Extracts PMID from `response.message["external-ids"]` where `id-type === "PMID"`.
6. If PMID found: updates row with `pmid = pmid` and `url = 'https://pubmed.ncbi.nlm.nih.gov/${pmid}/'`.
7. If PMID NOT found: logs `[SKIP] DOI ${doi} — no PMID in CrossRef` and adds to gap report array.
8. Adds 150ms delay between CrossRef requests (courtesy rate limiting).
9. Also updates any rows that HAVE pmid but url is empty/null: `url = 'https://pubmed.ncbi.nlm.nih.gov/${pmid}/'` (covers the 3 pre-existing PMIDs per D-05).
10. At end, prints summary: `Resolved: N, Skipped: M, Already had PMID: 3` and gap report if any.

Install dev dependencies first: `npm install --save-dev tsx dotenv`.

**Run the script:** `npx tsx scripts/backfill-pubmed.ts` — capture output, verify no errors.

**Step 1 — Create `scripts/backfill-youtube.ts` (per D-07, D-08):**
1. Loads env from `.env.local` using `dotenv.config({ path: path.join(process.cwd(), ".env.local") })`.
2. Creates TWO Supabase clients:
   - `agentDb = createClient(process.env.NEXT_PUBLIC_AGENT_SUPABASE_URL!, process.env.AGENT_SUPABASE_SERVICE_KEY!)` — for evidence_links
   - `cmsDb = createClient(process.env.NEXT_PUBLIC_CMS_SUPABASE_URL!, process.env.CMS_SUPABASE_SERVICE_KEY!)` — for manufacturer_youtube_transcript
3. Queries evidence_links: `SELECT id, url, snippet FROM evidence_links WHERE source = 'youtube' AND url NOT LIKE '%&t=%'` (skip rows already timestamped, per Research pitfall 7).
4. For each row:
   a. Extract `video_id` from url using regex: `url.match(/(?:v=|youtu\.be\/)([A-Za-z0-9_-]{11})/)?.[1]`
   b. Take first 40 chars of `snippet` as search token (escape SQL wildcards: `%`, `_`, `\`)
   c. Query CMS: `SELECT start_time, end_time FROM manufacturer_youtube_transcript WHERE video_id = '{video_id}' AND {text_column} ILIKE '%{searchToken}%' LIMIT 1`
   d. If match found: `start_time_int = Math.round(start_time)`, update evidence_links: `url = 'https://www.youtube.com/watch?v={video_id}&t={start_time_int}s'`
   e. If no match: try with first 20 chars (fallback). If still no match: log `[SKIP] evidence_link ${id} — no transcript chunk match for video ${video_id}` and add to gap array.
5. Print summary: `Timestamped: N, Skipped: M, Total YouTube rows: 6`.
