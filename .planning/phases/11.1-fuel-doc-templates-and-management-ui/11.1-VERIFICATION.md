### Data-Flow Trace (Level 4)

| Artifact | Data Variable | Source | Produces Real Data | Status |
|----------|---------------|--------|--------------------|--------|
| `app/dashboard/fuel-library/page.tsx` | `docs: FuelDoc[]` | `fetch('/api/fuel-docs')` → `listFuelDocs()` → `agentSupabase.from("gl_agent_fuel_documents").select("*")` | Yes — real DB query. Falls back to `SEED_FUEL_DOCS` if API returns empty array or errors. | FLOWING (with seed fallback) |
| `app/dashboard/fuel-library/[id]/page.tsx` | `doc: FuelDoc \| null` | `fetch('/api/fuel-docs/${id}')` → `getFuelDoc()` → `agentSupabase.from("gl_agent_fuel_documents")...single()` | Yes — real DB query. Falls back to SEED_FUEL_DOCS for seed- prefixed IDs. | FLOWING (with seed fallback) |

**Note on seed data fallback:** The UI uses `SEED_FUEL_DOCS` as a graceful fallback for two scenarios: (1) list page — if API returns an empty array, seeds are shown so the UI is demonstrable; (2) detail page — if the ID starts with `seed-`, the seed record is used without calling the API. This is an intentional design decision documented in the Plan 04 summary ("added seed fuel docs with real A360 content"). For seed doc IDs, edits silently fail (PATCH goes to API which won't find a seed- UUID). This is a known limitation, not a blocker for the phase goal.
