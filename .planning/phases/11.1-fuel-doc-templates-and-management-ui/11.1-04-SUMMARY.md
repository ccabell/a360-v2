# Phase 11.1 Plan 04: Fuel Doc Detail Page and Tabs Summary

Fuel doc detail page at `/dashboard/fuel-library/[id]` with 7 tabs (6 content sections + Raw JSON), inline edit mode per tab, and status promotion flow (draft → in_review → approved → active).

## Tasks Completed

### Task 1: Create detail page and all 6 tab components (d341c42)

Created 7 files following the agent detail page pattern (`app/dashboard/agents/[id]/page.tsx`).

**Detail page** (`app/dashboard/fuel-library/[id]/page.tsx`):
- `"use client"` with `useParams`, `useRouter`, `fetch` to `/api/fuel-docs/{id}`
- Header: product name (h1 text-xl font-semibold), FuelTypeBadge, ReviewStatusBadge, Edit/Done Editing toggle
- 7 tabs: Identity, Clinical, What to Say, What Not to Say, SOPs, Metadata, Raw JSON
- PATCH handler merges edits back via `/api/fuel-docs/{id}` then refetches
- Loading/error states with back navigation

**Tab components** (all follow the same prop interface: `doc`, `editing`, `onSave`):
- `IdentityTab`: fuel_type (read-only badge), patient_facing_name, one_line_positioning, type-specific field (package_goal/mechanism_summary/patient_language), audience checkboxes, patient_safe toggle, last_reviewed
- `ClinicalTab`: type-discriminated fields — combination (why_together, a_solves/b_adds, clinical_rationale), product (fda_indications, off_label_common, contraindications, does_not_solve), concern (underlying_cause, what_helps/what_does_not_help, treatment_sequence, expected_timeline, realistic_expectations)
- `WhatToSayTab`: combination (patient_education_summary, staff_close, staff_talking_points), product (key_talking_points, differentiators), concern (consultation_language, staff_talking_points)
- `WhatNotToSayTab`: combination (do_not_say.universal/pair_specific/practice_specific nested), product (do_not_say, do_not_claim), concern (do_not_say, do_not_promise)
- `SopTab`: shows "SOPs and preferences are practice-level fields" message when no practice data; renders full SOP + preference fields when practice data present
- `MetadataTab`: template_version, review_status with promote button (Submit for review / Approve / Activate based on current status), reviewed_by, timestamps

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] TypeScript discriminated union cast pattern**
- **Found during:** Task 1 verification (tsc --noEmit)
- **Issue:** Initial implementation cast `doc.content as Record<string, unknown>` then re-cast to `CombinationContent` etc., which TypeScript rejected (no sufficient overlap)
- **Fix:** Rewrote all tab components to use IIFE blocks after `doc.fuel_type` checks, allowing TypeScript to narrow `doc.content` to the correct type naturally. Added explicit return type annotation `(): Record<string, string>` on useState initializer to resolve union inference errors.
- **Files modified:** clinical-tab.tsx, what-to-say-tab.tsx, what-not-to-say-tab.tsx
- **Commit:** d341c42 (incorporated into single task commit)
