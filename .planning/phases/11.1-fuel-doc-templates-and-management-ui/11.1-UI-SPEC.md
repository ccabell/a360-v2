## Spacing Scale

Declared values (all multiples of 4). Uses Tailwind v4 CSS-variable token system — reference by Tailwind utility classes, not raw px values.

| Token | Value | Tailwind Class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, inline tight padding |
| sm | 8px | `gap-2`, `p-2` | Compact badge/tag spacing |
| md | 16px | `gap-4`, `p-4` | Default element and form field spacing |
| lg | 24px | `gap-6`, `p-6` | Card internal padding, section dividers |
| xl | 32px | `gap-8`, `p-8` | Panel and tab content padding |
| 2xl | 48px | `gap-12`, `py-12` | Major section breaks |
| 3xl | 64px | `gap-16`, `py-16` | Page-level vertical rhythm |

Exceptions: Touch targets for action icon buttons use minimum 44px hit area (apply `min-h-11 min-w-11` with `p-2` for visual size). This matches the Agent Manager toolbar button pattern.

## Typography

Uses shadcn/Tailwind v4 CSS variable tokens. Specify by Tailwind utility class.

| Role | Size | Weight | Line Height | Tailwind Classes | Usage |
|------|------|--------|-------------|-----------------|-------|
| Body | 14px | 400 (normal) | 1.5 | `text-sm font-normal leading-normal` | Table cells, dialog body, tab content prose |
| Label | 12px | 400 (normal) | 1.4 | `text-xs font-normal leading-tight` | Column headers, badge labels, helper text, `muted-foreground` |
| Heading | 16px | 600 (semibold) | 1.2 | `text-base font-semibold leading-tight` | Card titles, tab headings, dialog titles |
| Display | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` | Page title (`h1`) in list and detail views |

Rule: exactly 2 weights in use — 400 (normal) and 600 (semibold). No 500 or 700.

Source: Matches pattern observed in `app/dashboard/agents/page.tsx` — `font-medium text-foreground` for primary cell text, `text-xs text-muted-foreground font-mono` for secondary identifiers.

## Color

All color values expressed as CSS variable tokens. The existing OKLch palette from `app/globals.css` is the source of truth. Do not hardcode hex values — use Tailwind semantic class names.

| Role | CSS Variable | Tailwind Class | Usage |
|------|-------------|----------------|-------|
| Dominant (60%) | `--background` / `--card` | `bg-background`, `bg-card` | Page background, card surfaces, main content area |
| Secondary (30%) | `--sidebar` / `--muted` | `bg-sidebar`, `bg-muted` | Sidebar, filter bar, tab strip, muted panels |

Accent reserved for: (1) "Create Fuel Doc" primary button, (2) active sidebar nav highlight, (3) active tab indicator. Not used for status badges, row hover, or informational elements.

## Interaction Contracts

### List Page (`/dashboard/fuel-library`)

- `DataTable` renders one row per `gl_agent_fuel_documents` row.
- Columns: Name (product/pair/concern label), Type (`fuel_type` badge), Status (`review_status` badge), Template Version, Last Updated.
- Filter bar: filter by `fuel_type` (combination / product / concern) and `review_status` (draft / in_review / approved / active). Filters are additive (AND logic).
- Row click navigates to `/dashboard/fuel-library/{id}`.
- "Create Fuel Doc" button in toolbar opens `CreateFuelDocDialog`.
- Refresh button (icon-only, `RefreshCw` lucide icon) refetches the list. Matches Agent Manager refresh pattern.

### Create Dialog (`CreateFuelDocDialog`)

- Fields: `fuel_type` (Select, required), reference entity (product/pair/concern name — depends on type selection, required), optional display name override.
- On submit: POST to `/api/fuel-docs`, redirect to detail page on success, show error toast on failure.
- Cancel closes dialog with no side effects.

### Detail Page (`/dashboard/fuel-library/[id]`)

- Page header: fuel doc name (Display typography), `review_status` badge, back link to list.
- Tab strip below header with one tab per logical field group:

| Tab | Sections |
|-----|---------|
| Identity | `fuel_type`, `patient_facing_name`, `one_line_positioning`, `audience`, `patient_safe`, `last_reviewed` |
| Clinical | `why_together` (combination only), `clinical_rationale`, `evidence_backing` |
| What to Say | `patient_education_summary`, `staff_close`, `staff_talking_points` |
| What Not to Say | `do_not_say` (universal), `pair_specific_cautions` (combination only), `practice_notes` |
| SOPs | `sop` (practice-level only, no GL fallback), `preferences` (practice-level only), `room_requirements` |
| Metadata | `template_version`, `review_status` (with promote/approve/activate action), `reviewed_by`, `created_at`, `updated_at` |

- Edit mode: each tab has an "Edit" button that switches text display to form fields inline. Save applies PATCH to `/api/fuel-docs/{id}`. Cancel restores prior values. Unsaved changes trigger the unsaved-changes warning dialog on tab switch or navigation away.
- Raw JSON view: a "Raw JSON" toggle (icon: `Code` lucide icon) appears in the tab strip as the last tab. Shows `JsonViewer` for read-only, `CodeEditor` for edit mode.
- Practice override indicator: fields that have a practice override show a subtle `bg-muted` pill labeled "Practice override" beside the field label. GL fallback fields show no indicator (clean by default).

### Status Workflow

Status transitions are linear: `draft` → `in_review` → `approved` → `active`.

| Current Status | Action Button | Label |
|---------------|--------------|-------|
| draft | Primary | "Submit for review" |
| in_review | Primary | "Approve" |
| approved | Primary | "Activate" |
| active | No promote action | (already active) |

Demotion (e.g., active → draft) is not available in v1 of this UI.
