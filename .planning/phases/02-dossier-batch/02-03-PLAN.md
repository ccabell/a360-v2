<!-- Product dossier row structure (from DOSSIER_TEMPLATES.md) -->
Each product produces 5 rows in agent_reference_docs (extends_doc_id -> category doc):
  1. lens=clinical, doc_type=clinical_summary, audience=provider
     Sections: identity, mechanism, indications, candidacy, safety, outcomes
  2. lens=clinical, doc_type=technique_guide, audience=provider
     Sections: injection_planes, dosing_zones [gateway: range+pointer], technique_notes, planning, combination_sequencing
  3. lens=sales_education, doc_type=patient_education, audience=patient  *(PRIMARY)*
     Sections: patient_explainer, benefit_framing, combination_therapy, cost_benefit, maintenance_story
  4. lens=sales_education, doc_type=faq, audience=patient  *(PRIMARY)*
     Sections: objection_reframes, common_questions
  5. lens=deep_product, doc_type=deep_dive_playbook, audience=provider
     Sections: differentiation, mechanism_detail, evidence_summary, comparison_anchors

<!-- Structured emission spec (per STRUCTURED_EMISSION_ADDENDUM.md) -->
item_concerns: offering_id, concern_id, relevance (primary|secondary|adjunctive), treatment_role, is_fda_indicated, notes
item_body_areas: offering_id, body_area_id (most specific level), side, anatomy_specificity, notes
aliases: phrase (patient language), concern_id or body_area_id, normalized (generated)
does_not_solve: text[] on products table

<!-- Taxonomy audit mandate -->
For each product, audit concerns/body_areas/aliases. If something is missing, ADD it:
- New concern: aesthetic condition, clinical name, valid category enum, patient_description
- New body_area: slot into hierarchy with correct parent_id and level
- New alias: patient-language phrase pointing to existing concern/body_area
Log every addition for TAXONOMY_ADDITIONS.md
