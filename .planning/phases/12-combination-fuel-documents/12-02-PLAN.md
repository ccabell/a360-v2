<!-- From 12-RESEARCH.md "Content Generation Strategy" section -->

Review card field -> Canonical template field:
- "### Clinical Rationale" -> clinical_rationale, why_together
- "### Timing Guidance" -> timing_note, sequencing_note
- "### Patient Education Text" -> patient_education_summary
- "### Staff Talking Points" -> staff_close, staff_talking_points
- "## Evidence" section -> source_support_summary, evidence_level
- "**Same Session OK:**" in header -> same_session_ok
- "**Tier:**" in header -> (metadata only)

Fields that REQUIRE GENERATION (not in review cards):
- patient_facing_name: outcome-focused simple name (e.g., "Smooth & Restore")
- one_line_positioning: one sentence explaining the combination value
- ideal_candidate / not_ideal_candidate: extract from Gate results or generate
- A_solves / A_does_not_solve / B_adds: extract from Gate 3 (Limitation Coverage)
- top_objections: generate from universal matrix + pair-specific adaptation
- do_not_say.pair_specific: generate based on product categories
- maintenance_story: generate from timing data + cadence info
- rebooking_trigger: staff cue for rebooking conversation
- package_goal / concern_tags / downtime_note / next_visit_prompt: generate

Evidence level mapping:
- HIGH (PubMed + FDA) -> "strong"
- MODERATE (expert consensus + some data) -> "moderate"
- LOW (expert consensus only) -> "weak"
- Very limited studies -> "emerging"

Staff language quality checklist (10 items):
1. Sounds like provider-mediated education, not a sales script
2. Explains why both treatments are relevant to the patient's concern
3. Does not imply the patient "needs" more treatment
4. Avoids guarantees ("you will," "guaranteed," "always works")
5. Avoids unsupported clinical claims
6. Avoids fear-based language ("your skin will only get worse")
7. Uses patient-understandable outcome language (not jargon)
8. Preserves clinical accuracy
9. Includes realistic downtime/timing caveats
10. Could be said by a trained aesthetic consultant in a real consultation

    For each of the 5 canonical pairs (Botox Cosmetic + each of: Juvederm Vollure XC, Juvederm Voluma XC, Restylane Lyft, RHA Redensity, SKINVIVE by Juvederm):

    1. **Read the review card** from `REVIEW_QUEUE/pairings/{pair}.md`
    2. **Extract mapped fields** directly from the review card:
       - `clinical_rationale`: from "### Clinical Rationale" section (preserve the text, do not rewrite unless tone violates quality checklist)
       - `why_together`: synthesize from clinical rationale — 1-2 sentence mechanism summary
       - `timing_note`: from "### Timing Guidance" section
       - `sequencing_note`: extract sequencing-specific guidance from timing section
       - `patient_education_summary`: from "### Patient Education Text" section
       - `staff_close`: from "### Staff Talking Points" section — the closing recommendation language
       - `staff_talking_points`: from "### Staff Talking Points" section — the full conversational guidance
       - `same_session_ok`: from header "**Same Session OK:**" (boolean)
       - `source_support_summary`: from "## Evidence" section — list citations
       - `evidence_level`: map from Evidence Type header (HIGH->strong, MODERATE->moderate, LOW->weak)
       - `A_solves` / `A_does_not_solve` / `B_adds`: extract from Gate 3 (Limitation Coverage) in gate results

    3. **Generate missing fields** (not in review cards):
       - `patient_facing_name`: outcome-focused simple name describing what the combination achieves (e.g., "Smooth & Restore" for Botox + Vollure). Keep it simple, not branded/clever.
       - `one_line_positioning`: one sentence explaining combined value in patient-understandable terms
       - `ideal_candidate`: who benefits most (extract from Gate 1 Concern Overlap or generate)
       - `not_ideal_candidate`: who should NOT get this combination (extract from Gate 5 Safety or generate)
       - `package_goal`: the treatment outcome in 5-10 words
       - `concern_tags`: array of concern names from Gate 1
       - `downtime_note`: generate from timing guidance and product characteristics
       - `maintenance_story`: generate from timing guidance cadence info ("Botox every 3-4 months, filler every 12-18 months")
       - `rebooking_trigger`: staff cue language for follow-up ("At your 2-week follow-up, we can assess...")
       - `next_visit_prompt`: what to mention at the next appointment
       - `top_objections`: generate 3-5 objections per pair from the universal objection types (cost, "do I need both?", fear of overdone, downtime, "I'll think about it") adapted to the specific pairing. Each objection has: objection_type, patient_says, handling_language, do_not_say_in_response
       - `do_not_say.pair_specific`: 3-5 pair-specific things to avoid saying, based on the product categories (e.g., for neurotoxin+filler: never say "liquid facelift", never promise permanent results)

    4. **Apply quality checklist** — run each generated doc through the 10-item staff language checklist. Note any failures.

    5. **Format output** as a markdown file with each pair as a section containing the full canonical JSON. Include a quality checklist pass/fail summary per pair.

    CRITICAL RULES:
    - Content from review cards is the baseline. Do NOT discard or rewrite it unless it fails the quality checklist.
    - Do NOT introduce podcast attribution. No show names, episode names, or speaker names.
    - Expert consensus language is acceptable: "Expert consensus supports..." with no attribution.
    - Do NOT write to item_relationships table. Fuel docs are separate from relationship metadata.
    - Universal do-not-say list (12 items) goes in `do_not_say.universal` on every doc.
    - `practice_specific` in do_not_say is always empty array `[]` (practice fills this in via UI later).
