    ```sql
    -- Phase 09, Plan 02, Task 1: PubMed URL backfill
    -- Requirement: EVID-01
    -- Backfills url for evidence_links rows where source='pubmed' AND url IS NULL AND doi IS NOT NULL
    -- URL format: https://doi.org/{doi} (DOI resolver — redirects to publisher page)
    -- Alternative: If pmid is also available, prefer https://pubmed.ncbi.nlm.nih.gov/{pmid}/
    -- Idempotent: WHERE url IS NULL means re-running is safe
    ```

    2. The UPDATE statement — two passes:
    ```sql
    -- Pass 1: Rows with pmid — use PubMed direct URL (preferred)
    UPDATE evidence_links
    SET url = 'https://pubmed.ncbi.nlm.nih.gov/' || pmid || '/'
    WHERE source = 'pubmed'
      AND url IS NULL
      AND pmid IS NOT NULL;

    -- Pass 2: Rows with doi but no pmid — use DOI resolver
    UPDATE evidence_links
    SET url = 'https://doi.org/' || doi
    WHERE source = 'pubmed'
      AND url IS NULL
      AND doi IS NOT NULL
      AND pmid IS NULL;
    ```

    3. Trailing verification query:
    ```sql
    -- Verification: Should return 0 rows with NULL url for pubmed source
    SELECT id, doi, pmid, url
    FROM evidence_links
    WHERE source = 'pubmed'
      AND url IS NULL;
    ```

    The file must be idempotent (WHERE url IS NULL guard) and follow the same pattern as existing compile_sql files (header comment, UPDATE, verification SELECT).
