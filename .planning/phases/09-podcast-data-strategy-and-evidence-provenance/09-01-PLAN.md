    **## 1. Two-Layer Model**
    Define two layers:
    - **Research Layer**: Podcast-derived IDEAS and CONCEPTS. Used internally for knowledge discovery, enrichment planning, and research. Stored with anonymous identifiers only. Fields: concept_id (SHA-256 hash of normalized idea text, truncated to 12 hex chars, prefixed "EC-"), concept_text (the idea/concept in neutral language — no quotes, no attribution), discovery_source_type ("podcast_corpus"), discovery_date, pubmed_backup_doi (nullable — filled when PubMed corroboration found).
    - **Production Layer**: What appears in user-facing content. Sources: PubMed, FDA labels, medical society guidance, manufacturer data, expert consensus. Fields in item_relationships that are production-facing: clinical_rationale, patient_education_text, staff_talking_points, evidence_notes. Fields in agent_reference_docs that are production-facing: content (the dossier prose), evidence_links references.

    **## 2. Anonymous Identifier Scheme (per POD-02)**
    - Format: `EC-{12-hex-chars}` (e.g., EC-3a7f9b2c1d4e)
    - Generation: SHA-256 of lowercase, whitespace-normalized concept text, truncated to first 12 hex characters
    - Example: The idea "biostimulator + neurotoxin combination provides complementary mechanism coverage" -> normalize -> SHA-256 -> truncate -> EC-xxxxxxxxxxxx
    - Purpose: References like "expert consensus finding EC-3a7f9b2c1d4e" can appear in internal documentation without exposing speaker, show, or episode
    - Rule: The EC- ID NEVER appears in production-facing fields. It is for internal tracking and research-layer cross-referencing only.

    **## 3. Production Field Rules**
    Table listing every production-facing field and what is/isn't allowed:

    | Table | Field | Allowed Sources | Forbidden |
    |-------|-------|----------------|-----------|
    | item_relationships | clinical_rationale | PubMed citation, FDA label ref, "expert consensus supports..." | Speaker names, show names, episode IDs, "podcast experts say..." |
    | item_relationships | patient_education_text | Plain-language synthesis, PubMed-backed claims | Any podcast attribution |
    | item_relationships | staff_talking_points | Training guidance, evidence-backed selling points | Podcast quotes, speaker endorsements |
    | item_relationships | source_reference | PubMed DOI/PMID, FDA URL, "expert consensus" | Podcast episode URLs, show names |
    | agent_reference_docs | content | Synthesized prose backed by evidence_links | Raw podcast transcripts, speaker quotes |

    **## 4. Contamination Definition**
    A production field is "contaminated" if it contains ANY of:
    - Speaker names (e.g., "Dr. X says...")
    - Show names (e.g., "on the XYZ podcast...")
    - Episode identifiers (episode numbers, dates tied to shows)
    - Direct quotes attributed to podcast sources
    - The phrase "podcast experts" or "podcast consensus"
    - Any reference that would let a reader identify the specific podcast source

    What is NOT contamination:
    - Using an IDEA discovered via podcast, expressed in neutral language (e.g., "expert consensus supports biostimulator + neurotoxin combination")
    - Citing PubMed papers that corroborate a podcast-discovered idea
    - Referencing "expert consensus" generically (without podcast attribution)

    **## 5. Source Classification Matrix (EVID-02)**
    Create a separate SOURCE_CLASSIFICATION.md file with this matrix:

    | Source Type | Layer | Citable in Production? | Example |
    |-------------|-------|----------------------|---------|
    | PubMed | Production | Yes — DOI/PMID required | pubmed.ncbi.nlm.nih.gov/12345678 |
    | FDA Label | Production | Yes — URL required | accessdata.fda.gov/... |
    | Medical Society Guidance | Production | Yes — organization + year | ASPS 2023 practice advisory |
    | Manufacturer Data | Production | Yes — company + document | Allergan Botox PI |
    | Expert Consensus | Production | Yes — "expert consensus" (no attribution) | "Expert consensus supports..." |
    | Podcast Corpus | Research-only | NO — ideas only, via EC- ID | EC-3a7f9b2c1d4e |
    | Conference Presentation | Research-only | NO — unless published in proceedings with DOI | IMCAS 2024 talk |
    | Webinar | Research-only | NO — unless backed by published source | Industry webinar |
    | YouTube (clinical) | Research-only | NO — unless backed by published source | YouTube clinical demo |

    Include a "Decision Tree" section:
    1. Is the source PubMed, FDA, or medical society? -> Production-citable
    2. Is it podcast/conference/webinar/YouTube? -> Research-only
    3. Does the research-only source have a PubMed backup? -> Cite the PubMed backup in production
    4. No PubMed backup? -> Use "expert consensus" language in production, track via EC- ID internally

    **## Purpose**
    Step-by-step workflow for safely using the 630K+ podcast corpus as a research source. Podcasts are the richest source of aesthetic medicine intelligence. This workflow ensures podcast IDEAS fuel the knowledge base while podcast ATTRIBUTION stays out of production content.

    **## Prerequisites**
    - CMS Supabase access (project gjqicqldjgvrwmtkliie)
    - RPC function: match_podcast_chunks()
    - Understanding of EVIDENCE_MODEL.md two-layer model

    **## Workflow Steps**

    **Step 1: Generate Research Questions**
    Before any enrichment task, formulate specific questions. Example: "What combinations work well with Sculptra? What is the ideal timing between biostimulator and neurotoxin?"

    **Step 2: Mine the Podcast Corpus**
    Query match_podcast_chunks() with embedding vectors for each question. Collect the raw chunk results. At this stage, you see speaker names, show names, episode IDs — this is the research layer.

    **Step 3: Extract IDEAS, Not Quotes**
    From the raw chunks, extract the underlying clinical idea or concept. Transform:
    - BAD (quote): "Dr. Smith on AestheticsPro says Sculptra works great with toxin because the collagen stimulation complements the muscle relaxation"
    - GOOD (idea): "Biostimulator + neurotoxin combination provides complementary mechanism coverage: collagen remodeling addresses volume while muscle relaxation addresses dynamic lines"

    **Step 4: Generate Anonymous Identifier**
    For each extracted idea, generate an EC- ID per EVIDENCE_MODEL.md Section 2:
    1. Normalize the concept text (lowercase, collapse whitespace)
    2. SHA-256 hash
    3. Truncate to 12 hex chars
    4. Prefix with EC-

    **Step 5: Search for PubMed Corroboration**
    For each idea, search PubMed for published evidence that supports the same concept. Use match_pubmed_articles() or direct PubMed search.
    - If PubMed backup found: Record the DOI/PMID alongside the EC- ID
    - If no PubMed backup: The idea can still be used as "expert consensus" in production

    **Step 6: Write Production Content**
    When writing production-facing fields (clinical_rationale, patient_education_text, staff_talking_points, source_reference):
    - If PubMed backup exists: Cite the PubMed source. Example: "Combination of neurotoxin and HA filler showed superior outcomes (DOI: 10.xxxx/yyyy)"
    - If no PubMed backup: Use "expert consensus" language. Example: "Expert consensus supports biostimulator + neurotoxin combination for complementary mechanism coverage"
    - NEVER include: speaker names, show names, episode IDs, direct quotes, "podcast experts"

    **Step 7: Record Research Layer**
    In internal documentation (research notes, planning files, enrichment logs), the EC- ID can be used to trace back to the original podcast finding. The EC- ID links to the concept text and discovery metadata — but this NEVER appears in production SQL or user-facing content.

    **## Example: End-to-End**
    1. Question: "What combinations work well with Sculptra?"
    2. Corpus hit: [chunk from show X, speaker Y] "...Sculptra and Botox together because the collagen stimulation needs 6-8 weeks while the Botox kicks in immediately, giving patients visible improvement while waiting for the biostimulator..."
    3. Extracted idea: "Biostimulator + neurotoxin sequential combination addresses immediate visible improvement (neurotoxin) while biostimulator collagen remodeling develops over 6-8 weeks"
    4. EC- ID: EC-a8f3c2e19b74
    5. PubMed search: Found — "Combined use of poly-L-lactic acid and botulinum toxin A for facial rejuvenation" (PMID: 29876543)
    6. Production field: clinical_rationale = "Sequential neurotoxin + biostimulator combination provides immediate dynamic line improvement while collagen remodeling develops over 6-8 weeks (PMID: 29876543)"
    7. Research note: "EC-a8f3c2e19b74 — corroborated by PMID 29876543"

    **## Anti-Patterns (What NOT to Do)**
    - Do NOT copy podcast transcript text into clinical_rationale or staff_talking_points
    - Do NOT reference "multiple podcast experts" as a citation
    - Do NOT include speaker names in source_reference
    - Do NOT strip ALL podcast knowledge — podcasts are gold; use the IDEAS, remove the ATTRIBUTION
    - Do NOT panic and remove podcast-discovered concepts — the concept is valid, only the citation path changes

    **## Quick Reference Card**
    | Step | Input | Output | Goes Where |
    |------|-------|--------|------------|
    | Mine corpus | Research question | Raw chunks with speaker/show | Research layer only |
    | Extract idea | Raw chunk | Neutral concept text | EC- ID generation |
    | Find backup | Concept text | PubMed DOI/PMID or "none" | Production source |
    | Write content | Concept + source | Production-safe field text | item_relationships / agent_reference_docs |
