## Task 1 — Spot-check each in-scope PDF
For each of the 12 in-scope PDFs (see 00-CONTEXT.md D-02 for the list):
1. Open the PDF (page 1, one middle page, last page).
2. Confirm: content matches the claimed device/manufacturer; text is
   readable (not corrupted OCR); no leftover ManualsLib chrome (index pages,
   "Other Manuals" footers) from the automated cleanup.
3. Record page count as observed (cross-check against `MANIFEST.md`'s claimed
   count) and any issues found.

## Task 2 — Confirm gl-media hosting path
1. Check `C:\Projects\a360-studio\src\app\(library)\library\gl\page.tsx` and
   the storage-queue/upload-route files for a "Laser Manuals (LPOA)" browse
   root or any LPOA-specific path.
2. If found: use it, document the exact upload/URL pattern.
3. If not found: choose and document a fallback gl-media bucket path (e.g.
   `lpoa-manuals/{device-id}/{filename}.pdf`) — do not block on building new
   studio UI for this milestone.
3. Upload/verify the 12 in-scope PDFs are reachable at their gl-media URLs.

## Task 3 — Rights-distribution check
Per PDF, confirm it's a manufacturer-authored operator manual obtained via
ManualsLib (not paywalled/DRM'd), per D-06. Note any PDF that raises a
concern — surface it to Chris rather than self-approving.

## Task 4 — Write DEVICE_MANIFEST.md
One row per in-scope device: id, manufacturer, model, gl-media URL, page
count, source_quality (`operator_manual`), verified: yes, spot-check notes.
Append the 2 excluded devices at the bottom with in_scope: no and a one-line
reason, for traceability — do not verify them further.

## Task 5 — Findings note
Summarize for Chris: any PDFs that failed spot-check (and what P0 did about
it — re-clean, flag, or exclude), the gl-media path decided, and a restated
flag on D-03 (no device in the actual corpus has >1 manual, so R3's
multi-manual picker ships without a real multi-manual device to prove it this
milestone).
