Create `lib/api/fuel-docs.ts` following the exact pattern from `lib/api/agents.ts`. Import `agentSupabase` from `@/lib/supabase`. Import types from `@/lib/types/fuel-docs`.

Functions to implement:

1. `listFuelDocs(filters?: { type?: FuelDocType; status?: ReviewStatus; search?: string })`:
   - Query `gl_agent_fuel_documents` table via `agentSupabase`
   - Select `*`, order by `created_at` descending
   - If `filters.type` provided, filter by `source_type` (the existing column) matching the fuel_type value. Note: the existing DB column is `source_type`, not `fuel_type` — use `source_type` in the `.eq()` call.
   - If `filters.status` provided, filter by `review_status` column (will exist after migration, nullable before). If column doesn't exist yet, the query returns all rows (graceful).
   - If `filters.search` provided, use `.ilike("product_name", `%${filters.search}%`)`
   - Return data cast as `FuelDoc[]`

2. `getFuelDoc(id: string)`:
   - Query `gl_agent_fuel_documents` where `id` = param, `.single()`
   - Return cast as `FuelDoc`

3. `createFuelDoc(doc: { product_name: string; source_type: string; content: Record<string, any>; metadata?: Record<string, any> })`:
   - Insert into `gl_agent_fuel_documents`, `.select().single()`
   - Return cast as `FuelDoc`

4. `updateFuelDoc(id: string, updates: Partial<{ product_name: string; content: Record<string, any>; metadata: Record<string, any>; review_status: string }>)`:
   - Update `gl_agent_fuel_documents` where `id` = param, `.select().single()`
   - Return cast as `FuelDoc`

5. `deleteFuelDoc(id: string)`:
   - Delete from `gl_agent_fuel_documents` where `id` = param
   - Return void (no `.select()` needed)

6. `getResolvedFuelDoc(id: string, practiceId?: string)`:
   - First call `getFuelDoc(id)` to get the GL doc
   - If `practiceId` is provided, query `pl_agent_fuel_documents` where `gl_fuel_doc_id = id` AND `practice_id = practiceId`, `.maybeSingle()`
   - If practice override exists, merge: for each key in `pl.content`, override the corresponding key in `gl.content`. Set `sop` and `preferences` from pl row. Set `has_practice_override: true`.
   - If no practice override, return `{ ...glDoc, sop: null, preferences: null, has_practice_override: false }` cast as `ResolvedFuelDoc`
   - This implements the COALESCE pattern in TypeScript rather than SQL, matching the research recommendation.

All functions follow the try/catch + `if (error) throw error` pattern from `lib/api/agents.ts`. No additional error handling needed.
